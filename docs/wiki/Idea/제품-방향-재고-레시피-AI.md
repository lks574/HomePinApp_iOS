# 제품 방향 — 재고 + 레시피 + 온디바이스 AI

2026-06-12 아이디어 회의 결과. **방향**이며 코드/최종 아키텍처가 아니다(구현하며
화면·결정으로 구체화). 데이터 모델 결정은 `Decision/` 참조.

## 한 줄

> 말하거나 적으면 알아서 **위치·분류해 저장**하고, **말로 찾고**, **임박 재료로 뭐
> 해먹을지** 알려주는 로컬 앱.

## 정체성

집 물건 + 주방/식품 재고를 **위치에 핀**하고, **AI로 쉽게 넣고/찾고**, **레시피로
소비**한다. 서버 없는 로컬 앱.

## 킬러 루프

임박 재료 → AI 추천 → 요리 → 재고 차감.

## 탭 구성 (방향)

`홈 / 물건 / 레시피 / 설정` (4탭) + **NL ＋버튼**(말·글로 추가, 시그니처). 추가를
탭 한 칸으로 빼지 않고 어디서든 닿는 ＋ 로.

- **홈**: 검색 + 대시보드(임박 재료 **AI 추천 카드** 중심, 공간 바로가기, 최근).
- **물건**: 전체 목록 + 필터(위치·카테고리·태그·유통기한).
- **레시피**: 저장 + 재고 연동(보유/부족 표시, 임박 소비 추천, 부족분 쇼핑 리스트).
- **설정**.

## AI (Apple Foundation Models, 온디바이스·무료·오프라인)

자연어 → **구조화 드래프트** 한 엔진으로 **추가·자동분류·검색**을 모두 처리
(`@Generable` 구조화 출력). 자동 분류는 NL 추가의 부분집합.

- **추가**: "냉동실에 소고기 두 팩 넣었어" → `소고기 ×2 · 집>주방>냉동실 · 식품` 드래프트.
- **자동 분류·태깅**: 이름 → 카테고리·태그 추천(등록 마찰↓).
- **검색**: "곧 상하는 냉동실 고기" → 필터(`#Predicate`)로 변환.

핵심 규칙:
- 사용자 위치/카테고리에 **grounding**(자유 텍스트 표류 방지).
- **확인 드래프트 단계 필수** — AI가 채우고 사용자가 확인/수정 후 저장(소형 모델 오류 대비).
- **없으면 생성 / 있으면 기존 매칭 표시** — 확인 화면에서 *신규* vs *기존 매칭* 을
  **시각적으로 구분**(유사 중복 "냉동실 vs 냉동고" 방지).
- 여러 개 한 번에 파싱(배열).
- **텍스트 먼저, 음성 다음**(받아쓰기/SpeechAnalyzer → 같은 파서). 설계 때 계속 고려.
- **미지원 기기/한국어 한계 → 수동 fallback.** AI 는 거들기, 수동도 항상 됨.
