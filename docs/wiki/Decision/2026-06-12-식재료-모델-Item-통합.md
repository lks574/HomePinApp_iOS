# 2026-06-12 식재료 모델 — Item 통합

레시피가 재고와 연동되려면 "식재료" 를 어떻게 모델링할지 정한다.

## 결정: (a) Item 통합

- **별도 `Ingredient` 마스터를 두지 않는다.** 식재료 = `Item`(식품 카테고리).
- `Recipe`(추후)의 재료 라인은 **이름(+수량·단위) 텍스트**로 두고, 보유 여부는
  **Item 이름 매칭**으로 판단한다.

## 이유

- 단순함 — 모델·관계가 적고 초기 구현이 빠르다.
- 사용자 멘탈모델도 "재료도 그냥 내 물건" 에 가깝다.

## 트레이드오프 / 완화

- 이름 매칭은 **퍼지** — "소고기/쇠고기", "대파/파" 같은 동의어·표기차 매칭 한계.
  - 완화: AI grounding 시 **기존 Item 목록을 주고 정규화**(있으면 기존에 매핑,
    없으면 생성). 확인 단계에서 신규 vs 기존 구분.
- 정확도가 실사용에서 문제되면 **`Ingredient` 마스터 분리(b)로 승격** 가능
  (마이그레이션). 그때 Item 은 재고 인스턴스, Recipe 는 Ingredient 참조로 전환.

## 영향

- `Recipe` 모델 설계(추후) — 재료 라인 = 이름/수량/단위 + (선택) 매칭된 [[Item]].
- AI 추가/매핑 로직 — Item 이름 정규화·매칭이 품질 핵심.
