# 2026-06-12 위치·물건 데이터 모델

물건을 집 안 위치에 핀하는 로컬(SwiftData) 앱의 핵심 데이터 모델을 확정한다.

## 맥락

이전 구성이 중구난방이라 위치/물건 모델을 명확히 다시 잡는다. 도메인 개념은 4개:
공간(Space) · 구역(Area) · 세부위치(Spot) · 물건(Item). SwiftData 매크로를 적극 쓴다.

## 결정

### 1. 명시적 4모델 (고정 3단 위치 계층)

```
Space(공간)  ⊃  Area(구역)  ⊃  Spot(세부위치)        Item(물건)
```

재귀 트리(Location 단일 모델) 대신 **각 층을 별도 `@Model`** 로 둔다 — 타입 안전,
화면 단계화, 명확성 우선. 깊이는 3단 고정(가구는 Spot 이름/Area 로 흡수). 4단 이상
중첩이 반복적으로 필요해지면 재귀 트리 재검토.

### 2. 물건은 구역 필수 + 세부위치 선택

- 물건은 최소한 **구역(Area)** 에 속한다. 세부위치(Spot)는 있으면 더 정밀하게.
- "주방 어딘가"(area만) / "주방 냉동실 2번칸"(area+spot) 둘 다 표현.
- 불변식: `spot != nil` 이면 `item.area == spot.area`. 쓰기 로직에서 보장한다.

### 3. 삭제: 구조 cascade · 물건 nullify(보존)

- `Space → Area → Spot` 는 cascade (상위 삭제 시 하위 위치도 삭제).
- 위치 삭제 시 **물건은 지우지 않고 `area=nil`/`spot=nil` 로 보존** = "위치 미지정".
- **타입 디테일**: nullify 하려면 관계가 옵셔널이어야 하므로 `Item.area`·`Item.spot`
  은 **옵셔널**. "구역 필수" 는 DB 비옵셔널이 아니라 **앱 규칙(생성 시 강제)** 으로
  둔다. → 구역 삭제 시 그 물건들은 "위치 미지정" 버킷으로 남는다.

## 모델 / 매크로

공통(위치 3모델): `id`(`@Attribute(.unique)` UUID), `name`, `icon?`, `sortOrder`,
`createdAt`, `updatedAt`.

| 모델 | 부모(옵셔널) | 자식 `@Relationship` | 비고 |
| --- | --- | --- | --- |
| `Space` | — | `areas:[Area]` `.cascade` inverse `Area.space` | `#Unique([\.name])` |
| `Area` | `space:Space?` | `spots:[Spot]` `.cascade` inverse `Spot.area`<br>`items:[Item]` `.nullify` inverse `Item.area` | `#Unique([\.space,\.name])` |
| `Spot` | `area:Area?` | `items:[Item]` `.nullify` inverse `Item.spot` | `#Unique([\.area,\.name])` |
| `Item` | `area:Area?`(앱규칙 필수) · `spot:Spot?`(선택) | — | 필드 ↓ |
| `Category` | — | `items:[Item]` `.nullify` inverse `Item.category` | `#Unique([\.name])` |
| `Tag` | — | `items:[Item]` (N:N) inverse `Item.tags` | `#Unique([\.name])` |

### Item 필드 (확정)

- 코어: `name`, `quantity`(Int, 기본 1), `photoData: Data?`(`@Attribute(.externalStorage)`), `memo: String?`
- `expiresAt: Date?` — 유통/소비기한. `#Index([\.expiresAt])` 로 "곧 만료" 조회.
- `category: Category?` — to-one (분류 1개). `Category` 별도 `@Model`.
- `tags: [Tag]` — **N:N**. `Tag` 별도 `@Model`(재사용·이름변경·자동완성).
- 위치: `area: Area?`(앱규칙 필수) · `spot: Spot?`(선택)
- `createdAt`, `updatedAt`
- `@Transient` computed: `locationPath`, `isExpiringSoon` 등
- 바코드는 v1 제외(필요해지면 추가).

매크로 적극 사용:
- `@Model` ×4
- `@Attribute(.unique)` — 각 `id`
- `@Attribute(.externalStorage)` — `Item.photoData`(사진 외부 저장)
- `@Relationship(deleteRule:inverse:)` — 위 모든 엣지
- `#Unique` — 같은 부모 내 이름 중복 방지(한 공간에 "주방" 둘 금지)
- `#Index` — `Item.name` 등 검색/정렬 필드
- `@Transient` + computed — `Item.locationPath`("집 > 주방 > 냉동실"), `Area.itemCount`
