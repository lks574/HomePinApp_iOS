# 2026-06-12 네비게이션 / UI 구조

시안(`Downloads/음성 플로우.dc.html`, `장소·레시피.dc.html`)으로 확정. UI 우선 개발.

## 1. 탭바 (5-슬롯)

```
홈 │ 장소 │ [🎤 중앙] │ 레시피 │ 설정
```

- 4개 목적지 탭(홈/장소/레시피/설정) + **중앙 음성 추가 버튼(마이크)**. 중앙 버튼은
  탭(목적지)이 아니라 **NL 추가 입력**(텍스트필드+마이크 시트)을 띄운다.
- 둘째 탭은 "물건" 이 아니라 **장소**(공간별 둘러보기). 물건은 장소 상세에서 본다.
- 검색은 각 화면 상단 검색바(장소 상세 "…에서 찾기", 레시피 "재료 검색").

## 2. 위치 2레벨 UI ↔ 3레벨 모델 매핑 (중요)

시안은 위치를 **2레벨**(`장소 → 수납공간 → 물건`)로 보여준다. 우리 모델은 **3레벨**
([[Space]]⊃[[Area]]⊃[[Spot]]⊃[[Item]]). 매핑:

| UI 용어 | 모델 |
| --- | --- |
| (숨김) | `Space` — 단일 기본 "우리집" 자동 생성, UI 비노출 |
| **장소** | `Area` (주방·안방·냉장고·드레스룸…) |
| **수납공간** | `Spot` (서랍 1번째·옷장 상단·협탁…) |
| 물건 | `Item` |

- v1 은 **단일 Space**. "장소 추가" = 기본 Space 에 `Area` 추가.
- 멀티홈(별장 등)이 필요해지면 후속에 Space 스위처로 노출(모델은 이미 3레벨이라 무리 없음).
- `Item.area`(필수, 앱규칙)는 곧 "장소", `Item.spot`(선택)은 "수납공간".

## 3. 디자인 토큰 (시안 C)

`Shared/DesignSystem` 에 둔다.

- 배경: 페이지 `#E7E4DE`, 화면 `#F4EFE8`
- 강조(테라코타): `#C0603C`, 진한 `#A24E2F`
- 텍스트: 본문 `#2C2722`, 보조 `#6F665C` / `#8A8178` / 흐림 `#A89E92`
- 카드: 흰색, radius 18–20, 옅은 그림자
- 장소 배지: bg `#F3E2D8`, 텍스트 `#A24E2F`
- 재료 칩: 보유 `#F2EDE4`/`#6F665C` · 임박 `#FBEFE8`/`#C0603C` · **부족 점선** `#DCD2C5`(transparent) · 완비 `#E7F2EB`/`#1F8A5B`
- 폰트: Pretendard (번들은 후속, 우선 시스템 폰트)
- 탭바: 반투명 블러 `#F8F4EE`(82%), 중앙 마이크 = 떠 있는 테라코타 원

## 4. 화면 (UI 우선)

홈(대시보드) · 장소 목록(그리드) · 장소 상세(수납공간별 물건) · 레시피(임박 카드+추천)
· NL 추가 입력 시트(텍스트+음성) · 설정. **seed 데이터로 채워** 화면부터 만든 뒤 로직.
